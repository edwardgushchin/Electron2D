## Итог

Реализован внутренний shader source import baseline поверх import cache. `.e2shader` source assets импортируются в stable `shader.e2shader.json` cache artifact, optional sidecar `<shader>.e2import.json` задаёт target platforms, diagnostics сохраняют file/line/column/stage/target, а успешный iOS import получает заранее созданные vertex/fragment artifacts без runtime compilation.

## Проверенное поведение

- `ShaderSourceImporter` поддерживает `.e2shader`.
- Importer использует `CanvasShaderImportPipeline` и `ICanvasShaderCompiler`, поэтому сохраняет уже документированный формат `shader_type canvas_item;`, `vertex_entry` и `fragment_entry`.
- Optional sidecar `<shader>.e2import.json` задаёт список `targets`.
- Без sidecar importer использует default targets: `Windows`, `Linux`, `MacOS`, `Android`, `Ios`.
- Sidecar отслеживается как dependency import cache: изменение target list вызывает `DependencyChanged` reimport.
- `ShaderImportMetadataTextSerializer` пишет stable JSON с source, UID, `requiresRuntimeCompilation`, compiled stages и diagnostics.
- Diagnostics сохраняют `file`, `line`, `column`, `stage` и `target` из compiler output вида `path(line,column): error ...`.
- Для успешного `Ios` import artifact содержит vertex и fragment stages, `HasPrecompiledArtifacts(CanvasShaderTargetPlatform.Ios)` возвращает `true`, а `RequiresRuntimeCompilation` равно `false`.
- Public API не расширялся: public types осталось `47`.

## Заметки агента

Задача намеренно не добавляет public `ShaderFile`, public GPU shader handles, public `ResourceLoader` или compatibility layer. Importer остаётся внутренним механизмом для будущего editor/export pipeline и не расширяет целевой публичный API. Export packaging ещё не подключён к `shader.e2shader.json`; эта связь остаётся задачей export/tooling.
