## Итог

Введён baseline collision/material/body state целевого API для `0.1.0 Preview`: layer/mask helper methods на `CollisionObject2D`, public `PhysicsMaterial`, `PhysicsBody2D.PhysicsMaterialOverride`, validation и внутренние snapshots для future physics backend. Публичное имя restitution оформлено как `PhysicsMaterial.Bounce`, потому это целевой API.

## Проверенное поведение

- `SetCollisionLayerValue()`, `GetCollisionLayerValue()`, `SetCollisionMaskValue()`, `GetCollisionMaskValue()` используют целевую numbering convention `1..32`.
- Invalid layer/mask number даёт понятный `ArgumentOutOfRangeException`.
- `PhysicsMaterial` хранит `Friction`, `Bounce`, `Rough`, `Absorbent`.
- `Friction` и `Bounce` валидируются как finite values `>= 0`.
- `PhysicsMaterial` round-trip проходит через AOT-safe `ResourceObjectSerializer` metadata.
- `PhysicsBody2D.PhysicsMaterialOverride` синхронизируется во внутренний body-state snapshot.
- `RigidBody2D.GravityScale`, `Sleeping`, `CanSleep` синхронизируются во внутренний rigid-body snapshot во время `SceneTree.PhysicsFrame()`.
- Public API guard обновлён до `81` public type.
