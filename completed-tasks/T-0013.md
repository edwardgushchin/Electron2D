## Итог

Реализован Godot-like signal/callable baseline:

- `Callable`;
- `Error`;
- `ConnectFlags`;
- `Object.AddUserSignal(string signal)`;
- `Object.HasSignal(string signal)`;
- `Object.Connect(string signal, Callable callable, ConnectFlags flags = ConnectFlags.None)`;
- `Object.Disconnect(string signal, Callable callable)`;
- `Object.IsConnected(string signal, Callable callable)`;
- `Object.EmitSignal(string signal, params object?[] args)`.

## Проверенное поведение

- Multiple subscribers вызываются в порядке подключения.
- `Callable.From(Action)` и `Callable.From<T>(Action<T>)` работают для untyped/typed callbacks.
- `new Callable(Object target, string method)` вызывает public instance method target object.
- `Disconnect()` во время emission использует stable snapshot: текущий проход завершается, следующий проход учитывает disconnect.
- Unknown signal возвращает `Error.Unavailable`.
- Duplicate connection возвращает `Error.AlreadyExists`.
- Callback exception и mismatch аргументов возвращают `Error.Failed`, но не останавливают остальные callbacks.
- `Object` не добавляет .NET events в public surface.

## Отличия и будущие задачи

- До `StringName` signal/method names представлены `string`.
- До `Variant` signal arguments представлены `object?[]`, а `Callable.Call()` возвращает `object?`.
- `ConnectFlags.Deferred`, `Persist`, `OneShot` и `AppendSourceObject` пока объявлены, но runtime semantics остаются будущими задачами.
- `Callable.Bind()`, `Unbind()`, `CallDeferred()` и built-in lifecycle signals ещё не реализованы.
