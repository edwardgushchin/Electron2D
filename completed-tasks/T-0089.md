## Самодостаточное описание

Реализован Linux x64 glibc export baseline для `0.1.0 Preview`: внутренний planner строит deterministic package plan для `linux-x64`, Debug/Release, self-contained publish, renderer profile, `glibc` scope и desktop protocols `wayland`/`x11`. Отдельный verifier публикует шаблонный проект и запускает exported Linux executable на Linux host или через WSL, чтобы проверить reference scene и C# script lifecycle.

Реализация не добавляет public runtime API, не подписывает artifacts, не выполняет deploy и не публикует GitHub Release.

## Решения

- Linux export code размещён в `src/Electron2D/Export/Linux`, а не в `Core`.
- Planner не запускает внешние процессы: он строит проверяемый plan, а реальный publish/run выполняет отдельный verifier.
- Windows host запускает Linux verifier через WSL, потому что это даёт локальную проверку Linux executable без публикации релиза.
- CI запускает Linux verifier только на Linux runner, чтобы Windows/macOS jobs не пытались публиковать `linux-x64`.
