# T-0065: Реализовать `Tween`, easing, pause/resume и callbacks after completion

- Статус: завершена
- Завершена: 2026-06-21T12:57:00+03:00
- Приоритет: P1
- Зависимости: T-0064
- Домен: Анимация

## Исходная цель

Реализовать runtime baseline для `Tween`, чтобы `0.1.0 Preview` поддерживал deterministic property tweening, easing, pause/resume, stop/kill, manual step и callbacks after completion.

## Что сделано

- Добавлен публичный `Tween : RefCounted`.
- Добавлены `Tween.TransitionType` и `Tween.EaseType`.
- Добавлен публичный base type `Tweener`.
- Добавлены публичные `PropertyTweener`, `CallbackTweener` и `IntervalTweener`.
- `SceneTree.CreateTween()` создаёт valid tween и регистрирует его в tree processing.
- `Node.CreateTween()` создаёт tween через текущий `SceneTree` и привязывает его к node lifetime.
- `SceneTree.ProcessFrame()` теперь обрабатывает active tweens после node process callbacks и до draw traversal.
- `TweenProperty()` поддерживает property paths вида `Position` и `Position:X`, public property/field lookup, first-start initial value capture и interpolation для integer, floating-point, `Vector2`, `Color`.
- `TweenInterval()` добавляет time-only step.
- `TweenCallback()` вызывает `Callable` в sequence order.
- `Pause()`, `Play()`, `Stop()`, `Kill()` и `CustomStep()` реализуют deterministic time control.
- `finished` и `step_finished` signals испускаются при естественном завершении sequence и отдельных steps.
- `Kill()` invalidates tween без callbacks и без `finished`.
- Добавлены specification, implementation documentation, Wiki/API compatibility rows, release baseline update и public type guard updates.

## Важные решения

- Standalone `new Tween()` остаётся invalid: signals зарегистрированы, но tweeners можно добавлять только к tween, созданному через tree/node.
- Bound tween не двигается, пока bound node detached from tree; tween остаётся valid, а `IsRunning()` возвращает `false`.
- `Stop()` сбрасывает elapsed time и sequence index, но не откатывает target property и не заставляет property tweener заново захватывать initial value.
- Reflection lookup для property tweeners оставлен как runtime baseline с trim warning suppression; metadata-backed путь остаётся будущей задачей.
- Zero-duration callback после step, завершившегося ровно на границе frame delta, исполняется на следующем process/custom step, чтобы границы step completion были предсказуемыми.

## Ограничения

- Parallel tween groups, loops, subtween, await integration, physics process mode и scene-tree pause modes не входят в этот baseline.
- Editor timeline остаётся отдельной задачей анимационного домена.
- Reflection-based property lookup должен быть заменён на metadata-backed путь в будущей задаче AOT/tooling домена.
