## Итог

Реализован внутренний settings persistence baseline для `0.1.0 Preview`. Внутренний baseline означает слой кода движка и будущего редактора, который доступен тестам и toolchain, но не добавляет новый public runtime API.

## Что сделано

- Добавлен домен `src/Electron2D/Core/Settings`.
- Добавлен `Electron2DSettingsStore` для сохранения и загрузки project/user settings.
- Добавлен project settings JSON contract `Electron2D.ProjectSettings`, `formatVersion: 1`.
- `project.e2d.json` теперь содержит project identity, main scene, renderer profile, physics tick rate, `input.actions` и display/window defaults.
- `InputMap` actions сохраняются через существующий serializer input settings, поэтому keyboard, mouse button, gamepad button и gamepad axis bindings используют один формат.
- Добавлен user settings JSON contract `Electron2D.UserSettings`, `formatVersion: 1`.
- User settings сохраняют locale, last project path, recent projects и editor window state.
- Повреждённые project/user settings возвращают diagnostics и не применяют частичное состояние к `InputMap`, `DisplayServer` или `TranslationServer`.
- Шаблон `electron2d-empty` обновлён под новый `project.e2d.json`.

## Проверенное поведение

- Project settings round-trip сохраняет project identity, input actions, renderer profile, physics tick rate и display/window defaults.
- User settings round-trip сохраняет locale, recent projects и window state.
- Corrupt project settings возвращают `settings.malformed_json` и не заменяют текущий `InputMap`.
- Corrupt user settings возвращают `settings.malformed_json` и не меняют текущую locale.
- Runtime-only `InputEventAction` не сохраняется в project settings молча, а отклоняется с `FormatException`.
- Project template verifier проходит с расширенным `project.e2d.json`.

## Ограничения

- Public `ProjectSettings` API не добавлен.
- Public `Window` API не добавлен.
- Project Settings UI, Input Map UI и user settings path policy остаются задачами редактора/tooling.
