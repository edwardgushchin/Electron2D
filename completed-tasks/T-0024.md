## Итог

Введён internal CanvasItem render queue baseline без расширения public Godot-like API. Runtime получил deterministic сортировку canvas item commands, visibility filtering, effective modulate и contiguous batching с измеримым `DrawCallCount`.

## Проверенное поведение

- Public `CanvasItem`, `Node2D`, `Sprite2D` и `CanvasLayer` не добавлены; они остаются задачей `T-0026`.
- `CanvasItemRenderQueue` фильтрует invisible commands.
- Команды сортируются по layer, z-index, y-sort position, tree order и insertion order.
- Y-sort не пробивает z-index: item с меньшим z-index рисуется раньше.
- `CanvasItemRenderCommand.EffectiveModulate` вычисляется как `Modulate * SelfModulate`.
- Adjacent compatible commands с одинаковым `CanvasItemBatchKey` объединяются в один batch.
- Одинаковые batch keys, разделённые ordering barrier, остаются разными batches.
- `CanvasItemRenderPlan.DrawCallCount` равен количеству batches и даёт измеримый критерий batching.
- Public API compatibility baseline остался `28` публичных типов: новые CanvasItem queue-типы internal.

## Заметки агента

Batching не переупорядочивает commands ради уменьшения draw calls. Draw order остаётся первичным правилом; future public nodes должны передавать в queue уже вычисленные inherited visibility/modulate значения.
