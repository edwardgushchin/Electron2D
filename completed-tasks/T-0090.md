## Самодостаточное описание

Реализован macOS arm64 export baseline для `0.1.0 Preview`: внутренний planner строит deterministic package plan для `osx-arm64`, Debug/Release, self-contained `.app` bundle, Metal-backed desktop backend, x64 policy и user-provided signing plan. Отдельный macOS-only verifier публикует шаблонный проект, создаёт `.app` bundle с `Info.plist` и запускает `Contents/MacOS/Electron2D.Empty` на macOS arm64.

Реализация не добавляет public runtime API, не читает signing secrets, не выполняет deploy и не публикует GitHub Release.

## Ограничение проверки

`tools\Verify-MacOSExport.ps1` не запускался в текущей локальной сессии, потому что host - Windows. Полный publish/bundle/run требует macOS arm64 и выполняется через platform-specific CI gate.

## Решения

- macOS export code размещён в `src/Electron2D/Export/MacOS`, а не в `Core`.
- Planner не запускает внешние процессы: он строит проверяемый plan, а реальный publish/bundle/run выполняет отдельный verifier.
- User-provided signing представлен как deterministic `codesign` arguments в plan. Planner и verifier не читают реальные credentials.
- CI запускает macOS verifier только на macOS runner, чтобы Windows/Linux jobs не пытались публиковать `osx-arm64`.
