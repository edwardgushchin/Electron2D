## Итог

Реализован Godot-like deferred-call baseline:

- `Object.CallDeferred(string method, params object?[] args)`;
- `Callable.CallDeferred(params object?[] args)`;
- internal deterministic deferred queue;
- flush deferred queue после lifecycle/process/physics/input traversal;
- flush queued deletion после deferred queue;
- безопасный snapshot traversal для изменения children во время обхода.

## Проверенное поведение

- Deferred calls выполняются после текущего `_Process()` traversal.
- Порядок deferred calls соответствует порядку постановки в очередь.
- Deferred call, который ставит новый deferred call, выполняет вложенный вызов в той же idle-фазе.
- `QueueFree()` из `_Ready()` удаляет node после завершения lifecycle pass.
- `QueueFree()` из `_PhysicsProcess()` не прерывает sibling callbacks и удаляет subtree после physics traversal.
- `RemoveChild()` и `AddChild()` из `_Process()` не повреждают текущий traversal; новый child участвует в следующем frame pass.
- Ошибочный deferred callable не останавливает очередь.

## Отличия и будущие задачи

- До `StringName` method names представлены `string`.
- До `Variant` arguments представлены `object?[]`.
- `SetDeferred()` и deferred property assignment ещё не реализованы.
- Infinite deferred recursion guard будет частью будущей runtime diagnostics policy.
- Отдельного публичного idle-loop API пока нет.
