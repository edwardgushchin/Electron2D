# T-0036 [x] P0: Реализовать import cache отдельно от source assets

- Статус: завершена
- Коммит: `T-0036: реализовать resource import cache`

## Итог

Реализован внутренний import cache baseline для `0.1.0 Preview`: `.e2res` source assets обнаруживаются отдельно от cache root, импортируются в generated cache artifacts, переимпортируются при изменении source/dependency, сохраняют предыдущий валидный cache при ошибке и удаляются из cache при удалении source asset.

## Проверенное поведение

- `ResourceImportPipeline` рекурсивно сканирует `sourceRoot`, но не заходит в `cacheRoot`, `.electron2d`, `.git`, `.temp`, `bin`, `obj` и build/test output folders.
- `ResourceFileImporter` импортирует `.e2res`: валидирует `ResourceFileTextSerializer`, нормализует stable JSON и пишет cache artifact `resource.e2res`.
- `import-cache.json` хранит `source`, `uid`, `type`, `importer`, `sourceHash`, `cacheFiles` и dependency hashes.
- Повторный импорт без изменений возвращает `UpToDate`.
- Изменение dependency hash возвращает `DependencyChanged` и выполняет reimport.
- Невалидный source после успешного импорта возвращает `Failed`, но не перезаписывает предыдущий cache artifact и manifest.
- Удалённый source asset приводит к prune manifest entry и связанных cache artifacts.
- Public API не расширялся: import cache остаётся внутренним механизмом для тестов, будущего редактора и инструментов.
