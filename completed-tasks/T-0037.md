## Итог

Реализован внутренний PNG/JPEG texture import baseline поверх import cache. Image source assets импортируются в stable `texture.e2tex.json` cache artifact, sidecar `<image>.e2import.json` задаёт filtering, repeat, mipmaps, atlas regions и platform variants, а internal factory создаёт `Texture2D` и `AtlasTexture` instances из metadata.

## Проверенное поведение

- `TextureImageImporter` поддерживает `.png`, `.jpg`, `.jpeg`.
- PNG metadata reader извлекает width/height из `IHDR` и alpha из color type `4`/`6` или `tRNS`.
- JPEG metadata reader извлекает width/height из start-of-frame segment и фиксирует `hasAlpha=false`.
- Optional sidecar `<image>.e2import.json` задаёт `filter`, `repeat`, `mipmaps`, `atlas` и `platforms`.
- Sidecar отслеживается как dependency import cache: его изменение вызывает `DependencyChanged` reimport.
- `TextureImportMetadataTextSerializer` пишет stable JSON с сортировкой atlas regions и platform variants.
- `TextureImportResourceFactory` создаёт internal `Texture2D` и `AtlasTexture` instances из metadata.
- Public API не расширялся: public types осталось `47`.
