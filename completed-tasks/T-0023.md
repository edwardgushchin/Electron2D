## Итог

Введён internal SDL_GPU lifecycle baseline без расширения public Godot-like API. Runtime project теперь закрепляет managed dependency `SDL3-CS` версии `3.4.10.3`, а rendering domain содержит internal adapter/state machine для device creation, window claim, command buffer acquire/submit, resize/fullscreen/high-DPI diagnostics и shutdown.

## Проверенное поведение

- `src/Electron2D/Electron2D.csproj` содержит `PackageReference` на `SDL3-CS` `3.4.10.3`.
- `SdlGpuApi` вызывает SDL3-CS bindings для `SDL_CreateGPUDevice`, `SDL_ClaimWindowForGPUDevice`, `SDL_AcquireGPUCommandBuffer`, `SDL_SubmitGPUCommandBuffer` и `SDL_DestroyGPUDevice`.
- `SdlGpuRenderingBackend` проходит state machine `NotInitialized -> DeviceCreated -> WindowClaimed -> FrameOpen -> WindowClaimed -> Shutdown`.
- Resize сохраняет width/height/high-DPI scale и пишет событие `Resized`.
- Fullscreen change сохраняет flag и пишет событие `FullscreenChanged`.
- Device creation, command buffer acquire и другие lifecycle failures переводят backend в `Failed`, пишут `DeviceError` и выбрасывают понятный `InvalidOperationException`.
- `BeginFrame()` до initialize и повторный `BeginFrame()` при открытом frame запрещены.
- Public API compatibility baseline остался `28` публичных типов: новые SDL_GPU lifecycle типы internal.
