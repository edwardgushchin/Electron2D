## Итог

Реализован baseline sensors для `Area2D` из целевого API для `0.1.0 Preview`: область регистрирует built-in signal names, хранит overlap snapshots для тел и областей, учитывает filters и безопасно переживает `QueueFree()` внутри signal callback.

## Проверенное поведение

- `Area2D` регистрирует сигналы `body_entered`, `body_exited`, `area_entered`, `area_exited`.
- Добавлены public helper methods `GetOverlappingBodies()`, `GetOverlappingAreas()`, `HasOverlappingBodies()`, `HasOverlappingAreas()`, `OverlapsBody()`, `OverlapsArea()`.
- Managed baseline ищет пересечения через AABB активных `CollisionShape2D`.
- Учитываются `Monitoring`, `Monitorable`, `CollisionMask` monitoring area и `CollisionLayer` target object.
- Порядок сигналов зафиксирован: body enter, body exit, area enter, area exit; внутри группы nodes идут по `GetInstanceId()`.
- `QueueFree()` внутри overlap signal callback оставляет объект валидным на время callback, удаляет его после traversal и не ломает следующий physics frame.
- Public API не раскрывает Box2D.NET или другой backend handle.
