## Итог

Реализована внутренняя основа diagnostics без нового публичного API:

- `SceneTreeDiagnostic` хранит node context, callback, failure kind, exception, message и stack trace.
- `RuntimeUserCodeFailureKind` различает lifecycle, group call, deferred call и signal emission failures.
- Lifecycle callbacks, group calls, deferred calls и signal callbacks используют единую recover policy.
- Deferred queue переведена на per-`SceneTree` хранение, чтобы несколько `SceneTree` не перемешивали deferred calls.

## Проверенное поведение

- Lifecycle exception сохраняет diagnostic и traversal siblings продолжается.
- Deferred call exception сохраняет diagnostic и queue продолжает drain.
- Signal callback exception сохраняет diagnostic, `EmitSignal()` возвращает `Error.Failed` и вызывает следующие callbacks.
- Diagnostic содержит message и stack trace исходного user exception.
- Публичная поверхность runtime не изменилась: diagnostics остаётся внутренним механизмом.

## Отличия и будущие задачи

- Публичный диагностический API, уровни важности сообщений, панель вывода редактора и местоположение ошибки в исходном коде остаются будущими задачами.
- Signature mismatch без исходного user exception возвращает `Error.Failed`, но не обязан создавать diagnostic со stack trace.
