# T-0047: Реализовать безопасное editor-time выполнение `[Tool]` scripts

- Статус: завершена
- Завершена: 2026-06-21T13:10:00+03:00
- Приоритет: P1
- Зависимости: T-0045, T-0016
- Домен: C# scripting

## Исходная цель

Реализовать безопасный baseline для editor-time выполнения `[Tool]` scripts: отделить этот путь от обычного runtime traversal, изолировать exceptions и не вводить dynamic assembly load для AOT/export paths.

## Что сделано

- Добавлена спецификация `docs/specifications/scripting/tool-script-execution.md`.
- Добавлен internal `ToolScriptExecutionHost`.
- Добавлены internal `ToolScriptExecutionCallback`, `ToolScriptExecutionStatus` и `ToolScriptExecutionResult`.
- Host выполняет `_EnterTree()`, `_Ready()`, `_Process(delta)`, `_PhysicsProcess(delta)` и `_ExitTree()` только через явный call.
- Host разрешает execution только для registered `ScriptObjectTypeMetadata`, где `IsTool == true` и `IsToolExecutionSandboxed == true`.
- Для missing metadata, non-tool metadata и non-sandboxed metadata host возвращает structured skipped result и не вызывает callback.
- Callback exceptions не пробрасываются наружу, а возвращаются как status `ExceptionIsolated`.
- Host не предоставляет dynamic assembly loading capability и не принимает assembly/path/source inputs.
- В `ScriptObjectMetadataRegistry` добавлен fail-closed `TryGetByScriptType()`.
- Обновлены scripting docs и XML comments attributes, чтобы убрать запрещённые публичные формулировки из затронутого scripting-раздела.

## Важные решения

- Нового публичного API не добавлено: host остаётся internal contract для будущего editor/tooling layer.
- Runtime traversal не изменён: `SceneTree` не вызывает tool execution host.
- `[Tool]` attribute остаётся public marker, а фактическое выполнение требует explicit registered metadata.
- GUI editor run loop, hot reload, compile pipeline, external IDE workflow и загрузка пользовательских assemblies остаются отдельными задачами.

## Ограничения

- Host не запускает editor GUI run loop и не управляет scene preview.
- Host не компилирует C# и не загружает assemblies.
- Source generator для metadata остаётся отдельной задачей.
- Полная sandbox isolation process model не входит в baseline.
