## Итог

Реализован desktop baseline SDL input event mapping для `0.1.0 Preview`: SDL3-CS events остаются internal boundary, а пользовательский код получает Godot-like `InputEvent*` через существующий `SceneTree.DispatchInput()` и `Node._Input(InputEvent)`.

## Проверенное поведение

- `InputEventKey` получает SDL key down/up, `Pressed`, `Echo`, `Keycode`, `PhysicalKeycode`, `KeyLabel`, `KeyLocation` и modifier flags.
- SDL text input мапится в один или несколько `InputEventKey` с заполненным `Unicode`.
- `InputEventMouseButton` получает SDL mouse button down/up, position/global position, double click state и button index.
- SDL wheel мапится в `InputEventMouseButton` с `MouseButton.WheelUp`, `WheelDown`, `WheelLeft`, `WheelRight` и `Factor`.
- `InputEventMouseMotion` получает position/global position, relative/screen relative и Godot-like `MouseButtonMask`.
- Internal `SdlInputEventDispatcher` сохраняет порядок SDL event queue и dispatch-ит mapped events через `SceneTree.DispatchInput()`.
- Публичный `InputEventText` не добавлен, потому что он не входит в Godot-like API.
