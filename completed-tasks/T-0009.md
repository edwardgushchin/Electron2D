## Итог

Реализован первый lifecycle baseline целевого API без compatibility layer и без legacy Unity-like API:

- `Electron2D.Node`;
- `Electron2D.SceneTree`;
- `Electron2D.InputEvent`.

`Node` получил `Name`, `Parent`, `AddChild(Node)`, `RemoveChild(Node)`, `GetChildCount()`, `IsInsideTree()`, `GetTree()` и callbacks `_EnterTree()`, `_Ready()`, `_Process(double)`, `_PhysicsProcess(double)`, `_Input(InputEvent)`, `_ExitTree()`.

`SceneTree` создаёт `Root`, вводит его в дерево и предоставляет внутренние методы обхода для тестов и будущего runtime-кода редактора: `ProcessFrame(double)`, `PhysicsFrame(double)`, `DispatchInput(InputEvent)`.

## Проверенное поведение

- Добавление node к `SceneTree.Root` вызывает `_EnterTree()` и `_Ready()`.
- Subtree получает `_EnterTree()` сверху вниз и `_Ready()` снизу вверх.
- `_Ready()` вызывается для node один раз.
- `_Process()`, `_PhysicsProcess()` и `_Input()` идут сверху вниз.
- `RemoveChild()` вызывает `_ExitTree()` снизу вверх и очищает `Parent`/`GetTree()` у удалённого subtree root.
- Исключение из пользовательского callback сохраняется во внутреннюю диагностическую запись и не останавливает обход соседних node.
- Публичная поверхность runtime обновлена и остаётся целевой: legacy component/SpriteRenderer/Unity-like API не возвращались.

## Отличия и будущие задачи

- `SceneTree.Root` временно является `Node`; `Viewport`/`Window` root остаётся будущей задачей.
- Ownership, reparent, `QueueFree`, node paths, deferred calls и pause/process modes остаются в следующих задачах.
- Диагностический API пока внутренний; публичная модель диагностики остаётся в `T-0016`.
- Полный XML documentation gate и 100% coverage gate остаются в `T-0106` и `T-0108`.
