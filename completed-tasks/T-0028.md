## Итог

Реализован baseline immediate drawing для целевого API для `CanvasItem`: `_Draw()`, `QueueRedraw()`, `DrawLine()`, `DrawRect()`, `DrawCircle()`, `DrawPolygon()`, `DrawTexture()`, `DrawString()`, минимальные public `Font`/`HorizontalAlignment` и internal cached draw command submission.

## Проверенное поведение

- `QueueRedraw()` coalesces redraw: несколько вызовов до frame дают один `_Draw()` callback.
- Cached draw commands переиспользуются, пока item не queued for redraw.
- `Draw*` methods разрешены только во время `_Draw()` и бросают `InvalidOperationException` вне draw callback.
- Ошибки в `_Draw()` сохраняются как internal diagnostics и не ломают traversal.
- `CanvasSubmissionContext` отправляет line, rect, circle, polygon, texture и string commands в `CanvasItemRenderPlan`.
- `DrawTexture()` использует source rect по размеру `Texture2D`.
- `DrawString()` сохраняет `Font`, text, baseline position, `HorizontalAlignment`, width и font size, но не выполняет glyph layout.
- Golden-data тест фиксирует stable primitive command stream для line/rect/circle.
