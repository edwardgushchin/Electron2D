# T-0064: Реализовать `Animation`, `AnimationLibrary`, `AnimationPlayer`

- Статус: завершена
- Завершена: 2026-06-21T12:33:00+03:00
- Приоритет: P0
- Зависимости: T-0043
- Домен: Анимация

## Исходная цель

Реализовать runtime baseline для `Animation`, `AnimationLibrary` и `AnimationPlayer`, чтобы `0.1.0 Preview` поддерживал value tracks, method call tracks, discrete/interpolated values, queue playback и callbacks после завершения.

## Что сделано

- Добавлен публичный `Animation : Resource`.
- Добавлены `Animation.TrackTypeEnum`, `Animation.InterpolationTypeEnum` и `Animation.LoopModeEnum`.
- `Animation` хранит value tracks и method call tracks с deterministic key order.
- Value tracks поддерживают `TrackInsertKey()`, `TrackGetKey*()` и `ValueTrackInterpolate()`.
- `ValueTrackInterpolate()` поддерживает `Nearest` и `Linear` для `int`, `long`, `float`, `double`, `Vector2` и `Color`; остальные значения остаются дискретными.
- Method tracks поддерживают `MethodTrackInsertKey()` с `StringName` method name и `Variant[]` arguments.
- Добавлен публичный `AnimationLibrary : Resource`.
- `AnimationLibrary` поддерживает `AddAnimation()`, `RemoveAnimation()`, `RenameAnimation()`, `HasAnimation()`, `GetAnimation()` и stable `GetAnimationList()`.
- Добавлен публичный `AnimationPlayer : Node`.
- `AnimationPlayer` поддерживает libraries, default library с пустым именем, `library/animation` names, `RootNode`, `Autoplay`, `AssignedAnimation`, `CurrentAnimation`, `CurrentAnimationPosition`, `CurrentAnimationLength`, `SpeedScale`.
- Реализованы `Play()`, `Queue()`, `GetQueue()`, `ClearQueue()`, `Pause()`, `Stop()`, `IsPlaying()` и `Advance()`.
- `AnimationPlayer` применяет property values через `NodePath`, включая вложенные subnames вроде `Target:Position:X`.
- Method call tracks вызывают public methods через `Callable`.
- Завершение non-loop animation испускает сигнал `animation_finished` и запускает следующий queued animation.
- Добавлен internal process lifecycle hook, чтобы `AnimationPlayer` продвигался через `SceneTree.ProcessFrame()`.
- Добавлены specification, implementation documentation, Wiki/API compatibility rows, release baseline update и public type guard updates.

## Важные решения

- `AnimationPlayer.RootNode` по умолчанию равен `..`, поэтому track paths обычно разрешаются относительно parent node игрока анимации.
- Bare animation names ищутся в default library, а `library/animation` names - в named libraries.
- Missing track targets, properties или conversion failures не ломают scene traversal: такой track пропускается, остальные продолжают выполняться.
- Reflection lookup для property tracks оставлен как внутренний 0.1 baseline и снабжён trim warning suppression с пояснением; будущий metadata registry остаётся отдельной задачей.

## Ограничения

- `Tween` остаётся отдельной задачей анимационного домена.
- Editor timeline, blend tree, state machine editor, audio tracks, 3D transform tracks, skeletal animation, секции и capture mode не входят в этот baseline.
- Reflection-based property lookup должен быть заменён на metadata-backed путь в будущей задаче AOT/tooling домена.
