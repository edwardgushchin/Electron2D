## Итог

Реализован Godot-like `PackedScene` baseline:

- `PackedScene : Resource`;
- `PackedScene.Pack(Node path)`;
- `PackedScene.CanInstantiate()`;
- `PackedScene.Instantiate()`;
- `SceneTree.CurrentScene`;
- `SceneTree.ChangeSceneToPacked(PackedScene packedScene)`.

## Проверенное поведение

- `Pack()` сохраняет root node и owned descendants.
- Descendants без `Owner` не попадают в packed scene как runtime-only nodes.
- `Instantiate()` создаёт новый detached node tree и не переиспользует исходные node instances.
- Owner relationships переназначаются на cloned ancestors.
- Persistent groups сохраняются, transient groups не сериализуются в packed scene snapshot.
- Повторный `Pack()` заменяет ранее сохранённую сцену.
- `ChangeSceneToPacked()` откладывает замену active scene до следующего прохода `SceneTree`.
- Invalid packed scene возвращает `Error.InvalidParameter` и не меняет текущую сцену.

## Отличия и будущие задачи

- File-level scene serialization и загрузка по path остаются будущей задачей resource pipeline.
- Inherited scenes и editor-only instantiate modes не входят в baseline.
- Custom script state, exported properties и external node references ждут `Variant`/property database/resource pipeline.
