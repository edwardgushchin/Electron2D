# T-0043 [x] P0: Реализовать AOT-safe metadata для Inspector и serialization без `Reflection.Emit`, runtime IL generation и dynamic iOS code load

- Коммит: `T-0043: реализовать AOT-safe metadata`

## Итог

Реализован internal AOT-safe metadata baseline для custom `Resource` serialization и будущего Inspector. `ResourceObjectSerializer` больше не использует reflection fallback: он не ищет типы по загруженным сборкам, не обходит public properties и не создаёт custom resource через dynamic constructor lookup.

## Проверенное поведение

- `ResourceObjectMetadataRegistry` регистрирует `ResourceObjectTypeMetadata` по CLR type и serialized type name.
- `ResourceObjectTypeMetadata` хранит typed factory delegate и stable sorted property descriptors.
- `ResourceObjectPropertyMetadata` хранит serialized property name, CLR value type, getter/setter delegates и converter hooks.
- `ResourceObjectSerializer.Capture()` требует registered metadata и пишет property names из metadata, а не CLR property names.
- `ResourceObjectSerializer.Instantiate()` требует registered serialized type metadata и создаёт resource через typed factory delegate.
- Отсутствующая metadata даёт fail-closed `InvalidOperationException`.
- Property value converter покрывает scalar values, enum, nullable, arrays и dictionaries для registered descriptors.
- JSON serializers больше не выбирают AOT-опасный generic `JsonArray.Add<T>()` для serialization path.
- Public API не расширялся: metadata остаётся internal surface.

## Заметки агента

NativeAOT smoke по-прежнему выводит старые trim warnings по `Callable`, `SceneTree` и in-memory `PackedScene`. Они не относятся к новому metadata/serialization path и должны закрываться отдельными AOT задачами. Source generator для metadata пока не реализован; `0.1.0 Preview` получает runtime contract и ручную/test registration model.
