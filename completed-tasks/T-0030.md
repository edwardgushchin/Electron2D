## Итог

Реализован baseline для offscreen render target и восстановления texture resources после пересоздания GPU device.

Публичная Godot-like часть ограничена `Viewport.GetTexture()` и `ViewportTexture : Texture2D`. `ViewportTexture` создаётся движком, помечается как `ResourceLocalToScene`, отражает текущий `Viewport.Size`, сообщает наличие alpha и не раскрывает SDL_GPU handles.

Внутренняя часть для кода движка и тестов добавляет `TextureResourceRegistry.CreateRenderTarget()`, `TextureResourceUsage`, события `RenderTargetCreated`/`Restored` и `TextureResourceRegistry.RestoreAfterDeviceLoss()`.

## Проверенное поведение

- `Viewport.GetTexture()` возвращает один и тот же `ViewportTexture` для одного viewport.
- `ViewportTexture` наследуется от `Texture2D`, отражает текущий `Viewport.Size`, не имеет mipmaps и возвращает `false` из `IsPixelOpaque()` до появления public readback API.
- `TextureResourceRegistry.CreateRenderTarget()` создаёт descriptor с `TextureResourceUsage.RenderTarget`, сохраняет active handle и освобождает его через общий `Release()`.
- `RestoreAfterDeviceLoss()` переотправляет descriptors active sampled textures и render targets в новый `ITextureGpuApi`.
- `Rid` существующих ресурсов сохраняется при восстановлении.
- После успешного restore release идёт через новый adapter.
- Leak counter возвращается к `0` после release всех restored resources.
- Public API не получил SDL_GPU handles, public render-target handles или non-Godot методы.
