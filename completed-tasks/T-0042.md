## Итог

Реализован stress data stability gate для scene/resource pipeline. Добавлены автоматические проверки 100 save/load cycles, rename/move resource с сохранением UID, import cache rebuild и corruption diagnostics без silent data loss. В ходе задачи исправлен defect import cache: prune старого source path больше не удаляет cache artifact, который уже удерживается новым manifest entry после переноса ресурса с тем же UID.

## Проверенное поведение

- `SerializedResourceTextSerializer` и `SceneFileTextSerializer` выдерживают 100 последовательных `serialize -> deserialize -> serialize` циклов без изменения canonical JSON.
- Stress document покрывает arrays, dictionaries, enum, nullable и resource reference slots.
- При переносе `.e2res` на новый `res://...` path с тем же UID manifest переключается на новый source path, старый path попадает в prune report, а cache artifact остаётся на диске.
- Удаление cache root приводит к полному rebuild manifest/artifacts без изменения UID и без записи generated data в source root.
- Повреждённый `.e2res` получает status `Failed`, содержит malformed diagnostic, не перезаписывает старый валидный artifact и не портит manifest.
- Public API не расширялся: public types осталось `47`.
