# T-0075: Реализовать `Translation` resource, locale detection, manual switching, `Tr(string key)` и fallback locale

- Статус: завершена
- Завершена: 2026-06-21T13:45:00+03:00
- Приоритет: P0
- Зависимости: T-0035, T-0029
- Домен: Локализация и настройки

## Исходная цель

Реализовать runtime baseline локализации для `0.1.0 Preview`: `Translation` resource, определение начальной locale, ручное переключение locale, `Tr(string key)` и предсказуемый fallback для отсутствующих ключей. Видимый UI должен обновляться после смены locale.

## Что сделано

- Добавлен публичный `Translation : Resource`.
- Добавлен публичный `TranslationServer` как общий для процесса registry переводов и текущей locale.
- `Translation.Locale` нормализует locale names в стабильный вид с подчёркиванием между language и region.
- `Translation.AddMessage()`, `EraseMessage()`, `GetMessage()` и `GetMessageList()` поддерживают source key, optional context и стабильный список ключей.
- `TranslationServer.GetLocale()` получает текущую locale, а `SetLocale()` вручную переключает её.
- `TranslationServer.AddTranslation()`, `RemoveTranslation()`, `Clear()` и `GetLoadedLocales()` управляют загруженными переводами.
- `TranslationServer.Translate()` ищет перевод в порядке exact locale, base language, fallback `en`, исходный key.
- `Object.Tr(string message, string context = "")` делегирует перевод в `TranslationServer`.
- `Label` рисует `Tr(Text)` и обновляет cached draw commands после изменения locale или набора переводов на следующем processed frame.
- Добавлены спецификация, implementation documentation, Wiki/API compatibility rows и public type guard updates.

## Важные решения

- Fallback locale для `0.1.0 Preview` зафиксирован как `en`.
- Если ключ не найден, возвращается исходный key, чтобы интерфейс оставался предсказуемым и не превращался в пустую строку.
- `TranslationServer.Clear()` не сбрасывает текущую locale; это нужно для тестов, reload сценариев и будущего settings layer.
- Зарегистрированные `Translation` resources не копируются глубоко. Их лучше наполнять до регистрации или синхронизировать изменения снаружи.
- File import переводов и сохранение выбранной locale оставлены отдельным задачам ресурсов и настроек.

- File import переводов не входит в этот baseline.
- Project/user settings для сохранения выбранной locale остаются задачей `T-0076`.
