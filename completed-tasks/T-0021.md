## Итог

Реализован internal runtime serializer `VariantTextSerializer` для стабильного canonical JSON round-trip сериализуемого подмножества `Variant`. Публичный API не расширялся: serializer нужен будущему scene/resource pipeline и покрыт integration tests через `InternalsVisibleTo`.

## Проверенное поведение

- Exact JSON зафиксирован для `Nil`, `Bool`, `Int`, `Float` и `String`.
- Exact JSON зафиксирован для `Vector2`, `Vector2I`, `Rect2`, `Rect2I`, `Transform2D`, `Color`, `StringName` и `NodePath`.
- Nested `Array` сериализуется как JSON array of serialized Variant objects.
- `Dictionary` сериализуется как sorted entries `{"key":Variant,"value":Variant}` с ordinal sort по canonical JSON ключа.
- Повторная сериализация результата `Deserialize()` возвращает тот же canonical JSON.
- `Rid`, `Object` и `Callable` дают понятную `InvalidOperationException` при `Serialize()`.
- Malformed JSON, missing `type`, unknown `type` и runtime-only `type` в тексте дают `FormatException`.
- Public API compatibility baseline остался `25` публичных типов.

## Заметки агента

`Object`, `Resource`, `Node`, `Callable` и `Rid` остаются допустимыми runtime `Variant`, но не входят в stable portable data format `0.1.0 Preview`. File-level object/resource references должны появиться отдельной задачей вместе с resource pipeline.
