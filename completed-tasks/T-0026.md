## Итог

Реализован первый public canvas node baseline для `0.1.0 Preview`: `CanvasItem`, `Node2D`, `Sprite2D`, `CanvasLayer` и internal `CanvasSubmissionContext`, который собирает `Sprite2D` в `CanvasItemRenderPlan`.

## Проверенное поведение

- `CanvasItem` поддерживает `Visible`, `Modulate`, `SelfModulate`, `ZIndex`, `YSortEnabled`, `Show()`, `Hide()` и `IsVisibleInTree()`.
- `Node2D` поддерживает local/global `Position`, `Rotation`, `RotationDegrees`, `Scale`, `Transform`, `GlobalTransform`, `ToGlobal()`, `ToLocal()`, `Translate()`, `GlobalTranslate()`, `Rotate()` и `ApplyScale()`.
- `Node.Reparent(newParent, keepGlobalTransform: true)` сохраняет `Node2D.GlobalTransform`.
- `Sprite2D` поддерживает `Texture`, `Centered`, `Offset`, `FlipH`, `FlipV`, `RegionEnabled`, `RegionFilterClipEnabled`, `RegionRect`, `GetRect()` и `IsPixelOpaque()`.
- `CanvasLayer` поддерживает `Layer`, visibility и transform baseline.
- Internal submission учитывает layer, z-index, tree order, visibility, inherited `Modulate`, `SelfModulate`, transform, source/destination rects, flip flags и texture batch key.
- Public API остаётся Godot-like; `CanvasSubmissionContext`, render queue commands и texture handles не раскрыты наружу.

## Заметки агента

Public texture filter/repeat не добавлены в этой задаче: `CanvasItem` уже существует, но настоящий GPU drawing pipeline ещё не готов. Эти свойства стоит вводить вместе с реальным sampling/drawing behavior, чтобы не создать публичный API без эффекта.
