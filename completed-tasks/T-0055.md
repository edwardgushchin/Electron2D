## Итог

Введён baseline физических узлов целевого API для `0.1.0 Preview`: `CollisionObject2D`, `PhysicsBody2D`, `StaticBody2D`, `RigidBody2D`, `Area2D`, `CollisionShape2D`, `RayCast2D` и `Shape2D`. Узлы создают и освобождают RID через `PhysicsServer2D`, синхронизируют `GlobalTransform` во время physics frame и безопасно освобождаются через `QueueFree()` во время обхода `_PhysicsProcess()`.

## Проверенное поведение

- `StaticBody2D`, `RigidBody2D` и `Area2D` создают RID при входе в `SceneTree`.
- При `RemoveChild()`, `Free()` и `QueueFree()` RID освобождается через `PhysicsServer2D.FreeRid()`.
- Повторный вход узла в дерево создаёт новый RID.
- `CollisionObject2D.GetRid()` возвращает пустой `Rid`, когда узел не находится внутри дерева.
- Internal lifecycle hook в `Node` создаёт RID до пользовательского `_EnterTree()`, синхронизирует transform после пользовательского `_PhysicsProcess()` и освобождает RID после пользовательского `_ExitTree()`.
- `QueueFree()` внутри `_PhysicsProcess()` не ломает traversal: удаление и освобождение RID происходят после завершения текущего physics frame.
- `CollisionShape2D` хранит `Shape`, `Disabled`, `OneWayCollision`, `OneWayCollisionMargin`.
- `RayCast2D` хранит query-настройки и до реализации query backend возвращает пустой result state.
- Public API guard обновлён до `74` public types.
