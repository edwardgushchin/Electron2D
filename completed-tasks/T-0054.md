## Итог

Box2D.NET оформлен как кандидат физического backend для `0.1.0 Preview`, но не как зависимость основного runtime. Проверка вынесена в отдельный smoke-проект и PowerShell verifier, поэтому основной публичный API Electron2D продолжает скрывать Box2D-типы и работает через границы целевого API.

## Проверенное поведение

- Добавлен отдельный проект `tests/Electron2D.Tests.PhysicsBox2DSmoke`, который зависит от `Box2D.NET 3.1.654` и не подключает основной `Electron2D` runtime.
- Smoke-проект создаёт Box2D world, static ground body и dynamic circle body, выполняет warmup и measured ticks, проверяет движение тела под gravity и печатает стабильные метрики.
- Verifier `tools/Verify-Box2DPhysicsCandidate.ps1` запускает Release/JIT smoke, опционально публикует и запускает NativeAOT build, проверяет наличие документации по desktop и mobile ограничениям.
- Локально на `win-x64` проверены Release/JIT и NativeAOT: `AllocatedBytesPerTick=464.633` для Release/JIT и `AllocatedBytesPerTick=354.333` для NativeAOT.
- CI matrix дополнена обязательным запуском verifier с `-NativeAot`; это должно проверять Windows, Linux и macOS runners в рамках общего CI.
- Android/iOS Release/AOT gaps описаны как документированные ограничения: мобильная проверка не считается закрытой до появления real export pipeline для этих платформ.
- Основной `src/Electron2D/Electron2D.csproj` не получил `PackageReference` на Box2D.NET.
- Новые C# и PowerShell source-файлы созданы с полным MIT License header в стиле SDL, закреплённым в проекте.

## Заметки агента

Box2D.NET не становится публичным контрактом Electron2D в этой задаче. Он остаётся backend-кандидатом для последующей реализации `StaticBody2D`, `RigidBody2D`, shape APIs, queries и fixed timestep. Мобильные Android/iOS проверки не имитируются локально: они должны быть закрыты отдельными export задачами, где появятся реальные mobile toolchains.
