## Итог

Реализован resource file baseline для `0.1.0 Preview`: публичный `ResourceUid` из целевого API, stable `uid://` conversion, in-memory UID-to-path mapping, rename/move через `SetId()`, internal `.e2res` document model, external/internal references и stable LF JSON output.

## Проверенное поведение

- `ResourceUid.CreateIdForPath()` возвращает стабильный UID-кандидат для одного path.
- `ResourceUid.IdToText()` и `ResourceUid.TextToId()` выполняют round-trip `uid://`.
- `ResourceUid.AddId()` регистрирует UID/path mapping.
- `ResourceUid.SetId()` меняет path без изменения UID, поэтому ссылки переживают rename/move.
- `ResourceUid.PathToUid()`, `UidToPath()` и `EnsurePath()` проверены для зарегистрированных и незарегистрированных путей.
- `ResourceFileTextSerializer` выполняет round-trip `.e2res` с external/internal references.
- `.e2res` properties используют stable `Variant` serialization и не принимают runtime-only значения через `VariantTextSerializer`.
- Golden-data тест фиксирует exact JSON output с LF-переносами и стабильным порядком полей.
