## Итог

Реализован internal Android mobile GPU startup baseline. Runtime теперь имеет `SdlGpuStartupPolicy`, Android-safe `SdlGpuDeviceCreateInfo`, mobile smoke test, `Automatic` fallback на `Compatibility` и `FailIfUnavailable` failure path со structured startup result.

Публичный API не расширялся: mobile profile остаётся внутренней policy создания SDL_GPU device, а не новым public `RenderingProfile`.

## Проверенное поведение

- Android startup использует `SdlGpuDeviceCreateInfo.AndroidMobile(debugMode)`.
- В Android mobile create profile отключены optional Vulkan features: `ClipDistance`, `DepthClamping`, `IndirectDrawFirstInstance`, `Anisotropy`.
- Production `SdlGpuApi` создаёт device через `SDL_CreateGPUDeviceWithProperties`.
- Smoke test выполняет шаги `Texture`, `Pipeline`, `CommandBuffer`, `FirstSubmit` в стабильном порядке.
- `Automatic` policy при failure выбирает `CompatibilityRenderingBackend` и сохраняет root reason.
- `FailIfUnavailable` policy при failure выбрасывает `SdlGpuStartupException` с `SdlGpuStartupResult`.
- Startup result/log содержит backend, profile, fallback flag, GPU, driver, driver version и reasons.
- Public API compatibility count остался `47`.
