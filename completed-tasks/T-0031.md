# T-0031 [x] P1: Реализовать canvas shaders: import, SDL_shadercross compile, vertex/fragment stages и диагностику файл/строка

- Коммит: `T-0031: реализовать canvas shader import baseline`

## Итог

Реализован baseline canvas shader import для `0.1.0 Preview`: публичный `Shader` из целевого API, внутренний import pipeline, vertex/fragment stage compilation boundary через SDL_shadercross и diagnostics с file/line/column.

`Shader` хранит source code, возвращает `Shader.Mode.CanvasItem` и не раскрывает SDL_shadercross, SDL_GPU handles или compiled bytecode. `ShaderMaterial`, uniforms, texture samplers и engine-provided built-ins остаются отдельной задачей T-0032.

## Проверенное поведение

- `Shader.Code` хранит исходный текст и нормализует `null` в пустую строку.
- `Shader.GetMode()` возвращает `Shader.Mode.CanvasItem`.
- Public API не содержит `ShaderMaterial` до выполнения T-0032.
- `CanvasShaderImportPipeline` принимает source format `shader_type canvas_item;`, `vertex_entry <name>;`, `fragment_entry <name>;`.
- Header lines заменяются пустыми строками перед compiler boundary, чтобы line numbers diagnostics совпадали с source file.
- Import pipeline компилирует vertex и fragment stages для запрошенных target platforms.
- Compiler diagnostics вида `file(line,column): error ...` маппятся в structured diagnostic с severity, file path, line, column, stage и target platform.
- Успешный iOS import создаёт staged artifact и возвращает `RequiresRuntimeCompilation == false`.
- `SdlShaderCrossCompiler` использует `SDL3.ShaderCross`: DXIL для Windows, SPIR-V для Linux/Android и MSL для macOS/iOS.
