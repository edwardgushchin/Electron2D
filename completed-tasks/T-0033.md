## Итог

Реализован internal SDL_Renderer compatibility backend baseline. `CompatibilityRenderingBackend` теперь строит `SdlRendererFramePlan` из `CanvasItemRenderPlan`: спрайты, UI/text, immediate primitives, `DrawTexture()` и tile-like texture copies превращаются в детерминированный SDL_Renderer-compatible command stream.

Публичный API не расширялся. Это не compatibility layer для старого Electron2D API, а внутренняя подготовка fallback renderer profile.

## Проверенное поведение

- `CreateFramePlan()` возвращает backend name, profile, supported features, draw-call estimate, commands и limitation IDs.
- `Texture`, `Line`, `Rect`, `Circle`, `Polygon` и `String` canvas commands мапятся в SDL_Renderer-compatible operations.
- `Sprite2D`, `Label`, `DrawString()`, `DrawTexture()`, `DrawLine()`, `DrawRect()`, `DrawCircle()` и `DrawPolygon()` покрыты reference scene.
- `Compatibility` продолжает объявлять `Sprites`, `Animation`, `TileMap`, `Ui`, `Text`, `Primitives`, `Camera`, `Clipping` и `StandardBlendModes`.
- `Compatibility` не объявляет `RenderTargets`, `CustomShaders`, `ShaderMaterial`, `MultiPass`, `AdvancedBlending` и `PostProcessing`.
- Ограничения backend зафиксированы как stable IDs.
- Golden-data snapshot использует LF-only text output и стабилен между платформами.

## Ограничения

- Реальное создание `SDL_Renderer` и передача command stream в SDL3-CS ещё не реализованы.
- Screenshot из реального SDL window ещё не реализован.
- Public `TileMapLayer` не добавлялся; текущий backend принимает tile-like texture copies как обычные texture commands.
- `Shader` и `ShaderMaterial` остаются недоступными в `Compatibility` renderer profile.
