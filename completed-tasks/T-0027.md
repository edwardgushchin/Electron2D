## Итог

Реализован baseline целевого API для `Camera2D` и `Viewport`, текущей 2D camera, camera transform в internal sprite submission, pixel snapping и internal presentation plan для resize/high-DPI сценариев.

## Проверенное поведение

- `Camera2D` поддерживает `Enabled`, `IgnoreRotation`, `Offset`, `Zoom`, `MakeCurrent()`, `ClearCurrent()`, `IsCurrent()`, `GetTargetPosition()`, `GetScreenCenterPosition()` и `GetScreenRotation()`.
- `Camera2D.Zoom` отклоняет non-finite и zero компоненты.
- `Camera2D.Align()`, `ForceUpdateScroll()` и `ResetSmoothing()` являются документированными no-op для текущего baseline.
- `Viewport` поддерживает `Size`, `CanvasTransform`, `Snap2DTransformsToPixel`, `Snap2DVerticesToPixel`, `GetCamera2D()` и `GetVisibleRect()`.
- `SceneTree.Root` публично остаётся `Node`, но фактический root object теперь является `Viewport` с именем `root`.
- Internal `CanvasSubmissionContext` применяет viewport/camera transform и pixel snapping к `Sprite2D` render commands без mutation node properties.
- Internal `ViewportPresentationSettings` строит plan для `Disabled`, `CanvasItems`, `Viewport`, fractional/integer scaling и high-DPI framebuffer.
