# T-0041 [x] P0: Реализовать serializer для сцен, ресурсов, arrays, dictionaries, enums, nullable и пользовательских `Resource`

- Статус: завершена
- Коммит: `T-0041: реализовать scene resource serialization`

## Итог

Реализован внутренний file-level serializer baseline для сцен, ресурсов и переносимых property values. Добавлены stable JSON documents `Electron2D.SerializedResource` и `Electron2D.SceneFile`, typed property value model, custom `Resource` round-trip, arrays, dictionaries, enums, nullable и отдельные resource reference slots для внешних и внутренних ресурсов.

## Проверенное поведение

- `SerializedPropertyValue` поддерживает `Variant`, `Array`, `Dictionary`, `Enum`, `Nullable` и `Resource` reference slots.
- `SerializedResourceTextSerializer` пишет stable JSON resource document с UID, type, path, external/internal references и properties.
- `SceneFileTextSerializer` пишет stable JSON scene document с nodes, parent/owner ids, persistent groups, properties и resource references.
- `ResourceObjectSerializer.Capture()` создаёт document из public get/set properties custom `Resource`.
- `ResourceObjectSerializer.Instantiate()` восстанавливает custom `Resource` через parameterless constructor и сохраняет arrays, dictionaries, enum и nullable значения.
- `load -> modify -> save -> load` для scene document сохраняет values и references.
- Public API не расширялся: public types осталось `47`.

## Заметки агента

Задача намеренно не добавляет public `ResourceLoader`/`ResourceSaver` и не подключает file-level serializer к public `PackedScene`. AOT-safe metadata registry без reflection остаётся отдельной задачей `T-0043`. Resource references сохраняются как slots, но automatic reference resolution при `ResourceObjectSerializer.Instantiate()` ещё не реализован.
