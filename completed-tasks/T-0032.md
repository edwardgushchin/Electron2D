# T-0032 [x] P1: Реализовать `ShaderMaterial`, uniforms, texture samplers и engine-provided built-ins

- Коммит: `T-0032: реализовать ShaderMaterial baseline`

## Итог

Реализован baseline shader materials для `0.1.0 Preview`: public Godot-like `Material`/`ShaderMaterial`, supported uniforms, `Texture2D` sampler parameters, fail-closed validation reserved canvas built-ins и stable internal JSON snapshot для будущего resource pipeline.

`CanvasItem.Material`, real GPU binding uniforms/samplers, default texture parameter hints, global uniforms, per-instance uniforms и visual shader editor не входят в T-0032.

## Проверенное поведение

- `Material` наследуется от `Resource`.
- `Material.NextPass` хранит следующий pass material для будущего renderer ordering.
- `Material.RenderPriority` принимает только диапазон `-128..127`.
- `ShaderMaterial` наследуется от `Material`.
- `ShaderMaterial.Shader` хранит ссылку на `Shader`.
- `SetShaderParameter()` и `GetShaderParameter()` работают с `StringName`, чувствительны к регистру и возвращают nil для отсутствующего параметра.
- Запись nil `Variant` удаляет сохранённый параметр.
- Поддержаны shader parameter values: `Bool`, `Int`, `Float`, `Vector2`, `Color`, `Transform2D` и `Texture2D` sampler.
- Unsupported `Variant` types, обычные `Object`, `Callable`, `Rid`, строки, path/name values, коллекции, rect/integer-vector values отклоняются через `ArgumentException`.
- Canvas built-ins вроде `TIME`, `TEXTURE` и `texture_sdf` нельзя записать как user shader parameter.
- Internal `ShaderMaterialParameterTextSerializer` пишет stable JSON с `format = "Electron2D.ShaderMaterialParameters"` и `version = 1`.
- Public API count вырос до `47` типов: добавлены `Electron2D.Material` и `Electron2D.ShaderMaterial`.
