# T-0050 [x] P1: Реализовать gamepad lifecycle, mapping, axes, buttons и haptics/vibration

## Итог

Реализован desktop gamepad baseline для `0.1.0 Preview`: public API получил типы `JoyAxis`, `JoyButton`, `InputEventJoypadButton`, `InputEventJoypadMotion`, методы `Input` для connected devices, axis/button state и vibration state, а `InputMap` получил button/axis bindings с persistence в internal project settings serializer.

## Проверенное поведение

- Lifecycle-события подключения и отключения обновляют process-wide state `Input.GetConnectedJoypads()`, `Input.GetJoyName()` и `Input.IsJoyKnown()`.
- Button events доставляются в `_Input()` как `InputEventJoypadButton` и обновляют `Input.IsJoyButtonPressed()`.
- Axis events доставляются в `_Input()` как `InputEventJoypadMotion`, нормализуются в диапазон `-1.0..1.0` и обновляют `Input.GetJoyAxis()`.
- `InputMap` сопоставляет `InputEventJoypadButton` по `ButtonIndex`, а `InputEventJoypadMotion` по `Axis` и знаку `AxisValue`.
- Axis bindings учитывают deadzone и корректно отпускают action при возврате значения внутрь deadzone.
- Runtime state keys включают device id, поэтому одинаковый binding может удерживаться несколькими устройствами независимо.
- `StartJoyVibration()` и `StopJoyVibration()` безопасно работают с unsupported или unknown devices как no-op.
- Integration tests для stateful input выполняются последовательно, чтобы frame-local input transitions не сбрасывались параллельным `SceneTree.ProcessFrame()`.
