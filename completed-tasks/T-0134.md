## Самодостаточное описание

По уточнению пользователя `Core` нужен, но только для вещей, напрямую относящихся к ядру движка. Корневые домены должны быть крупными: мелкие подсистемы вроде input, settings, text и UI не должны лежать ни в `Core`, ни отдельными root-папками.

Namespace не должен следовать папкам. Runtime source сохраняет форму публичного API: основные типы находятся в `Electron2D`, коллекции - в `Electron2D.Collections`.

## Решения

- `Core` не удалён: он оставлен для базового ядра.
- Физическая папка не создаёт namespace. Добавление нового namespace требует отдельной задачи и API-обоснования.
- Корневой layout должен оставаться крупным: новые мелкие подсистемы добавляются внутрь `Assets`, `Graphics`, `Runtime`, `Physics`, `Export` или `Core`, если они действительно относятся к ядру.
- `T-0088` снята с in-progress до production-кода, чтобы Windows export не смешивался с layout refactor.
