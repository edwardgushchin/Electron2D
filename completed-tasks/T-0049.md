# T-0049: Реализовать `InputMap`, action persistence, deadzones и API `IsActionPressed`, `IsActionJustPressed`, `GetActionStrength`, `GetVector`

- Статус: завершена
- Завершена: 2026-06-21T14:05:00+03:00
- Приоритет: P0
- Зависимости: T-0048, T-0075
- Домен: Ввод

## Исходная цель

Реализовать action-level ввод для `0.1.0 Preview`: `InputMap`, сохранение action settings, deadzones, `Input.IsActionPressed()`, `Input.IsActionJustPressed()`, `Input.GetActionStrength()` и `Input.GetVector()`.

## Что сделано

- Добавлен публичный `InputEventAction : InputEvent`.
- Добавлен публичный process-wide `InputMap`.
- Добавлен публичный process-wide `Input`.
- `InputMap` поддерживает `AddAction()`, `EraseAction()`, `HasAction()`, `GetActions()`.
- `InputMap` поддерживает `ActionSetDeadzone()` и `ActionGetDeadzone()`.
- `InputMap` поддерживает `ActionAddEvent()`, `ActionEraseEvent()`, `ActionEraseEvents()`, `ActionGetEvents()` и `EventIsAction()`.
- `Input` поддерживает `IsActionPressed()`, `IsActionJustPressed()`, `GetActionStrength()` и `GetVector()`.
- `SceneTree.DispatchInput()` обновляет `Input` перед вызовом `_Input(InputEvent)`, чтобы пользовательский callback видел актуальный action state.
- Simultaneous bindings учитываются отдельно: release одного binding не отпускает action, пока другой binding остаётся active.
- Deadzone применяется к direct action events и `GetVector()`.
- Добавлен internal `InputMapProjectSettings` для deterministic JSON round-trip action settings без публичных file I/O методов на `InputMap`.
- Обновлены specification, implementation documentation, GitHub Wiki source, public API guard и input mapping documentation.

## Важные решения

- Persistence не вынесена в публичный API: сохранение остаётся внутренним serializer layer для будущего project settings workflow.
- `exactMatch` добавлен как совместимый параметр, но в этом baseline зарезервирован для будущей проверки modifier/device distinctions.
- `InputEventAction` служит direct action event для тестов, tooling и будущей runtime automation.
- `EventIsAction()` для keyboard/mouse проверяет совпадение binding независимо от pressed-state; pressed/released semantics применяются только при обновлении runtime state.
- Gamepad, touch/mobile input и UI focus pipeline оставлены отдельным input backlog.
