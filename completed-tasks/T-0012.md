## Итог

Реализован baseline групп целевого API:

- `Node.AddToGroup(string group, bool persistent = false)`;
- `Node.RemoveFromGroup(string group)`;
- `Node.IsInGroup(string group)`;
- `Node.GetGroups()`;
- `SceneTree.GetFirstNodeInGroup(string group)`;
- `SceneTree.GetNodeCountInGroup(string group)`;
- `SceneTree.GetNodesInGroup(string group)`;
- `SceneTree.HasGroup(string name)`;
- `SceneTree.CallGroup(string group, string method, params object?[] args)`.

Persistent metadata хранится internal и проверяется integration-тестами через friend assembly.

## Проверенное поведение

- Добавление/удаление groups работает без дубликатов.
- Persistent flag не сбрасывается повторным non-persistent add.
- `SceneTree` group queries возвращают только nodes inside tree.
- `GetNodesInGroup()` возвращает nodes в hierarchy order.
- `RemoveChild()` исключает subtree из group queries, но сохраняет локальное membership.
- `CallGroup()` вызывает public matching method в hierarchy order и игнорирует nodes без метода.

## Отличия и будущие задачи

- До `StringName` group/method names представлены `string`.
- До `Variant` `CallGroup()` использует `params object?[]`.
- `CallGroupFlags`, deferred/reverse group calls, `NotifyGroup()` и `SetGroup()` остаются будущими задачами.
- Persistence пока не сериализуется в scene files до появления `PackedScene`.
