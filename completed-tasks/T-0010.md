## Итог

Реализован baseline hierarchy для целевого API для `Node`:

- публичный `Parent` property удалён из runtime surface;
- parent читается через `GetParent()`;
- `Owner` хранится отдельно от parent и может ссылаться только на ancestor;
- добавлены `GetChild(int)`, `GetIndex()`, `MoveChild()`, `Reparent()`, `IsAncestorOf()`;
- добавлен `QueueFree()` и `Object.IsQueuedForDeletion()`;
- `SceneTree` выполняет queued deletion flush после runtime traversal.

## Проверенное поведение

- Public API `Node` не содержит `Parent`, но содержит `GetParent()` и `Owner`.
- Child order, отрицательный `GetChild()` index, `GetIndex()` и `MoveChild()` работают согласованно.
- `AddChild()` и `Reparent()` запрещают parent cycles.
- `Owner` принимает только ancestor и сбрасывается при detach/reparent, если прежний owner больше не ancestor.
- `Reparent()` переносит node между parent/tree/orphan состояниями и сохраняет owner, если owner остаётся ancestor.
- `QueueFree()` безопасен при повторных вызовах, удаляет subtree после текущего traversal и не прерывает sibling callbacks.
- `RemoveChild()` отсоединяет subtree без удаления и позволяет добавить его снова.
