## Итог

Реализован внутренний TTF/OTF font import baseline поверх import cache. Font source assets импортируются в stable `font.e2font.json` cache artifact, sidecar `<font>.e2import.json` задаёт fallback fonts и SDF/bitmap policy, а fallback font files участвуют в dependency tracking и cache invalidation.

## Проверенное поведение

- `FontImporter` поддерживает `.ttf` и `.otf`.
- `FontMetadataReader` читает sfnt signature `0x00010000` как `Ttf` и `OTTO` как `Otf`.
- `FontMetadataReader` читает `name` table records family/style/full/PostScript name.
- Optional sidecar `<font>.e2import.json` задаёт `fallbacks` и `rasterization`.
- Default rasterization policy: `Bitmap`, `baseSize=16`, `sdfSpread=0`.
- `Sdf` policy сохраняет `baseSize` и `sdfSpread` в metadata.
- Fallback font files отслеживаются как dependencies: изменение fallback file вызывает `DependencyChanged` reimport у основного font source.
- `FontImportMetadataTextSerializer` пишет stable JSON с сортировкой fallback paths.
- `FontImportResourceFactory` создаёт internal `Font` instance из metadata.
- Public API не расширялся: public types осталось `47`.

## Заметки агента

В задаче намеренно не добавлен public `FontFile`: импорт работает как внутренний pipeline и не расширяет пользовательский API без отдельной Godot-like API спецификации. SDF/bitmap policy пока metadata-level; glyph rasterization, atlas generation и SDL_ttf native file loading остаются следующими задачами.
