## Самодостаточное описание

Задача относится к домену «Export pipeline и платформы». Цель - реализовать общий export preset model и toolchain validation. До задачи единого export preset format не было, а платформенные export-задачи требовали общей serializable основы и проверяемых ошибок для отсутствующих SDK/toolchain/signing references.

Результат ограничен общим baseline: модель presets, стабильный JSON round-trip, загрузка с fail-closed diagnostics и validator окружения. Signing, deploy, публикация и platform-specific package build не выполняются.

## Что сделано

- Добавлен внутренний домен `src/Electron2D/Core/Export/`.
- Добавлен формат `export_presets.e2export.json`:
  - `format`: `Electron2D.ExportPresets`;
  - `formatVersion`: `1`;
  - `presets`: массив presets.
- Presets сохраняются в deterministic order по `name`.
- Duplicate preset names fail closed с diagnostic `E2D-EXPORT-PRESET-0001`.
- Toolchain validator проверяет .NET SDK, Android SDK/NDK, Xcode и signing references без чтения секретов.
- Diagnostic messages не раскрывают `credentialReference`.
- Public API не расширен: весь baseline остаётся внутренним механизмом для будущих editor/CLI/export tasks.

## Решения

- Реализация не добавляет public export API, потому что текущий релизный шаг нужен как внутренняя основа для будущих editor/CLI задач.
- Signing references хранятся только как ссылки. Секреты, certificates, tokens и passwords не читаются и не записываются.
- Validator не запускает build, signing, deploy, публикацию или platform package builder.
