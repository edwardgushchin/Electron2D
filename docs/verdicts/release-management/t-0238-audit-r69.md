BLOCKERS:

* B1

  * Что не так: Если `SystemClipboardTextAccess.TrySetText(clipboardSentinel)` возвращает `false`, ordinary submit больше не принимает system clipboard, но затем вызывает browser clipboard fallback с `staleClipboardText = null`. В этой ветке `TryReadClipboardResult` не может отличить старый clipboard text от результата текущей copy-кнопки и принимает любой текст, который вернул `navigator.clipboard.readText()`. Captured `navigator.clipboard.writeText()` argument проверяется только после неудачного `readText()`, а не как обязательное доказательство источника при отсутствии sentinel.
  * Почему это важно: r69 должна закрыть r68 B1 — запретить принятие старого clipboard report как verdict текущей отправки. Сейчас этот запрет действует только для system clipboard с установленным sentinel и для browser read, который вернул сам sentinel. Но при неустановленном sentinel старая browser clipboard строка может пройти strict report validation и быть сохранена как новый внешний verdict. Это особенно заметно на non-Windows платформах, где `SystemClipboardTextAccess.TrySetText` всегда возвращает `false`, но browser `navigator.clipboard.readText()` всё ещё может вернуть старое содержимое буфера.
  * Что исправить: Если sentinel не установлен, нельзя принимать `navigator.clipboard.readText()` как самостоятельный source. В таком случае нужно либо требовать captured `navigator.clipboard.writeText()` argument от текущей copy-кнопки, либо сначала установить sentinel через browser clipboard API, либо завершать polling-cycle локальной диагностикой/retry. Старый browser clipboard text не должен быть достаточным доказательством результата copy-click.
  * Как проверить исправление: Добавить regression, где sentinel installation failed, copy-click выполнен, `navigator.clipboard.readText()` возвращает заранее лежащий валидный report, а captured `writeText()` отсутствует или отличается. Ordinary submit должен отказать и не сохранить report. Добавить positive-test, где sentinel недоступен, но captured `writeText()` содержит Markdown текущего assistant response, и принимается именно captured value.
