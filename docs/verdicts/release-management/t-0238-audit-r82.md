* B1

  * Что не так: Browser copy-event fallback остаётся слишком широким. Если `event.clipboardData` не содержит текста, код читает глобальное выделение через `document.getSelection()` / `window.getSelection()`, затем весь текст активного элемента через `activeElement.value` или `activeElement.textContent`. Он не проверяет, что selection или active element принадлежат текущему assistant response, временному copy payload текущей copy action или DOM-поддереву нового сообщения.
  * Почему это важно: Ordinary submit должен сохранять Markdown именно из текущего нового ответа ассистента после клика по `button[data-testid="copy-turn-action-button"]` этого ответа. В r82 coordinate click и DOM click целятся в ту же текущую copy-кнопку, но copy-event capture всё ещё остаётся fallback-источником результата. Если Clipboard API capture, `event.clipboardData.setData(...)`, system clipboard и browser clipboard не дали новый Markdown, текущий fallback может принять произвольный старый выделенный текст или полный value активного поля как captured Markdown. Если такой текст выглядит как валидный final report и содержит текущие `metadata.taskId` / `metadata.iteration`, downstream validator примет его как verdict текущего запуска. Это нарушает proof-of-source contract ordinary submit.
  * Что исправить: Сделать copy-event fallback source-aware. Принимать selected/active text можно только при проверяемой связи с текущим assistant response или временным copy payload текущей copy action. Полный `active.value` / `active.textContent` без непустого selection range принимать нельзя. Без `event.clipboardData`, Clipboard API payload, pre-load/late captured payload, `DataTransfer.setData` capture, system clipboard sentinel proof, browser read sentinel proof или проверяемого current-response selection команда должна падать с диагностикой, а не сохранять report.
  * Как проверить исправление: Добавить regression, где copy listener установлен, current assistant copy button clicked, `event.clipboardData.getData(...)` пустой, а `document.getSelection()` или `activeElement.value` содержит валидный stale report, не связанный с текущим assistant response; ordinary submit должен отказать. Добавить positive regression для реального `execCommand` fallback, где selection принадлежит текущему assistant response или временному copy payload текущей кнопки. Сохранить regressions на coordinate click + DOM click по текущей copy-кнопке и на `DataTransfer.prototype.setData('text/plain', markdown)` capture.
