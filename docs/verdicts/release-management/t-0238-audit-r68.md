* B2

  * Что не так: Если sentinel установлен, но после copy-click browser `navigator.clipboard.readText()` возвращает сам sentinel, код принимает этот текст как candidate и не проверяет captured `navigator.clipboard.writeText()` argument. Если captured `writeText()` argument есть, его нужно сверять и использовать как доказуемый результат текущей copy-кнопки, а не только после полного failure `readText()`. Если ни system clipboard, ни captured value не доказывают current response, запуск должен падать локальной диагностикой, а не принимать sentinel как report candidate.
