BLOCKERS:

* B1

  * Что не так: Ordinary polling считает любой `null` от `CopyLatestAssistantMessageMarkdownAsync` после `IsGeneratingAsync == false` устойчивой недоступностью copy-кнопки. Но `CopyLatestAssistantMessageMarkdownAsync` возвращает `null` не только когда copy-кнопка отсутствует у уже появившегося нового assistant response, а также когда на странице ещё нет требуемого числа сообщений или последний assistant response текущей отправки ещё не появился. В результате команда может завершиться `E2D-BUILD-AUDIT-SUBMIT-ORDINARY-COPY-UNAVAILABLE` до появления нового ответа ассистента.
  * Почему это важно: control-r83 B1 требовал bounded diagnostic именно для состояния «новое ассистентское сообщение уже появилось, генерация завершена, минимальный счётчик сообщений достигнут, но copy-кнопка недоступна». Текущее исправление расширяет это состояние на обычное ожидание начала ответа. Для реального ChatGPT-submit задержка между отправкой user message и появлением assistant response может быть больше 30 секунд; такой запуск будет ошибочно классифицирован как broken copy action, хотя ответ ещё не готов.
  * Что исправить: Разделить состояния ordinary polling. `CopyLatestAssistantMessageMarkdownAsync` должен возвращать типизированный результат: например `NoCurrentAssistantYet`, `CopyButtonMissing`, `CopiedMarkdown`, `CopyActionFailed`, либо отдельный метод должен сначала доказать, что текущий assistant response уже появился и находится после user message текущей отправки. Таймер `ordinaryCopyButtonMissing` можно запускать только после подтверждения, что новый assistant response существует, generation завершена и именно copy-кнопка текущего response отсутствует. До этого команда должна продолжать обычное ожидание ответа или завершаться только общим timeout.
  * Как проверить исправление: Добавить regression, где `IsGeneratingAsync` возвращает `false`, новое user message уже есть, но нового assistant response ещё нет / message count меньше `messageCountBeforeSend + 2` дольше `OrdinaryCopyFailureStableAge`; ordinary polling не должен выбрасывать `E2D-BUILD-AUDIT-SUBMIT-ORDINARY-COPY-UNAVAILABLE`. Отдельный regression должен сохранить требуемое поведение control-r83: когда current assistant response уже есть, generation завершена, но copy-кнопка устойчиво отсутствует, команда падает с `E2D-BUILD-AUDIT-SUBMIT-ORDINARY-COPY-UNAVAILABLE`.
