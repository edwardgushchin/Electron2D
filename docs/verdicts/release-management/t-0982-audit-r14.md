* B1

  * Что не так: start-boundary проверка всё ещё безусловно принимает whitespace перед `repoRoot`. На POSIX whitespace может быть обычным символом имени файла, поэтому `repoRoot` может находиться внутри более длинного absolute path-token-а. Например, при `repoRoot = /home/user/repo` строка `/opt/prefix /home/user/repo/logs.txt` является одним POSIX absolute path-token-ом: сегмент `prefix ` содержит пробел, а `/home/user/repo` находится дальше внутри пути, а не является отдельным repo-root token-ом. Текущий код примет пробел перед candidate как безопасную левую границу, увидит `/` после repo-root как child separator и заменит середину пути на `/opt/prefix <repo>/logs.txt`.
  * Почему это важно: r14 scope и документация требуют сохранять embedded-root/non-repository machine paths как `E2D-BUILD-AUDIT-ABSOLUTE-PATH` blockers. Документация текущего пакета прямо говорит, что path separator перед opening delimiter/candidate на той же строке делает replacement недопустимым даже при whitespace внутри POSIX filename segment-а. В текущем коде эта current-line path-separator проверка применяется к opening delimiter, но не применяется к прямому whitespace перед candidate. После замены исходный repo-root исчезает из текста, и `ValidateMachineLocalPathText` уже не может заблокировать unsafe evidence.
  * Что исправить: применять full-token/current-line path check не только для opening delimiter, но и для whitespace-boundary. Практически: если перед `repoRoot` стоит whitespace, нужно проверить, не является ли этот whitespace частью POSIX path-token-а с path separator-ом ранее на той же строке. Надёжнее заменить локальные delimiter-эвристики на выделение полного surrounding path-token-а и заменять только token, который равен exact repo-root или нормализуется в safe child path внутри repoRoot.
  * Как проверить исправление: добавить focused regression через production `audit package` на POSIX/non-Windows: imported preflight `output.txt` содержит `/opt/prefix {slashRepoRoot}/logs.txt` и вариант с tab/space-containing segment. Ожидаемый результат — отказ `E2D-BUILD-AUDIT-ABSOLUTE-PATH`, а не ZIP с `/opt/prefix <repo>/logs.txt`. Позитивные cases вида `path: {slashRepoRoot}/logs.txt`, exact root и safe child path должны продолжать проходить.
