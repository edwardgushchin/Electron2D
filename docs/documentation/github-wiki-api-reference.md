## Назначение

Публичная API reference Electron2D должна генерироваться и публиковаться в отдельный GitHub Wiki repository:

```text
https://github.com/edwardgushchin/Electron2D.wiki.git
```

Основной репозиторий не должен хранить Wiki pages как отслеживаемый исходный код и не должен создавать локальный сайт. Каталог `.github/wiki/` зарезервирован только под игнорируемый локальный клон `Electron2D.wiki.git`, чтобы агент и CI работали с тем же расположением Wiki. В основном репозитории остаются только generator, verifier, спецификация и документация процесса.

## Источники данных

Генератор обязан брать данные из двух проверяемых источников:

- compiled runtime assembly `src/Electron2D/bin/Debug/net10.0/Electron2D.dll`;
- XML documentation file, созданный из текущих C# XML comments при build.

Ручной список типов не допускается как основной источник API reference, потому что он быстро расходится с public surface.

Для корневого контракта совместимости этот документ является потребителем generated source of truth, а не вторым перечнем public API. Точные имена типов, членов и marker-атрибутов должны приходить из `data/api/electron2d-api-manifest.json`, XML documentation и Wiki `API-Compatibility.md`; доменный текст описывает только процесс генерации, проверки и публикации Wiki.

## Структура GitHub Wiki

GitHub Wiki repository должен содержать:

- `Home.md` - входная страница Wiki;
- `_Sidebar.md` - короткая навигация без полного списка public types;
- `_Footer.md` - короткая версия и источник генерации;
- `API-by-Category.md` - вход в API reference по доменам;
- `API-*.md` - страницы доменов API;
- `API-Reference.md` - индекс публичных типов;
- `*.md` - по одной странице на каждый public type с коротким именем без повторяющегося префикса `API-Electron2D`;
- `API-Compatibility.md` - generated compatibility/status report из manual API profile и generated manifest.

Сгенерированные страницы должны иметь marker-комментарий:

```markdown
<!-- Generated by eng/Electron2D.Build update wiki. Do not edit by hand. -->
```

`API-Compatibility.md` тоже является generated page с этим marker. Страница строится по текущему exported runtime surface и его решениям из generated manifest, содержит строку `Generated from data/api/electron2d-public-api-profile.json`, верхнюю навигацию к `Home`, `API-by-Category`, `API-Reference` и самой странице compatibility, legend статусов `Supported`, `Deferred`, `Unsupported`, `Unapproved` и секцию `Current Public Runtime Surface`. Profile-only решения, для которых ещё нет exported type, не получают Wiki type page. Их можно проверить по одному через `e2d api compare-godot <type>`: команда напрямую читает решение из `data/api/electron2d-public-api-profile.json`, а `data/api/electron2d-api-manifest.json` использует только для экспортированности и parity evidence. Отдельная ручная таблица будущих API-типов не публикуется.

## Содержимое страниц типов

Для каждого public type страница должна включать:

- верхнюю навигацию `Home`, `API by Category`, `Complete API Index`, `API Compatibility`;
- metadata table с full name, namespace, kind и category;
- summary;
- декларацию типа;
- блок `Godot 4.7 C# profile compatibility`, построенный из machine-readable API manifest;
- remarks, thread-safety, since и see also, если они есть в XML documentation;
- таблицу public members, объявленных этим типом;
- для каждого member: kind, C# signature, summary, remarks, parameters, type parameters, returns, value, exceptions, thread-safety, since и see also, если эти данные есть в XML documentation.

Generator не должен обещать поведение, которого нет в XML documentation: если соответствующего XML-тега нет, раздел не выводится.

## Проверка синхронизации

Целевой verifier и generator для GitHub Wiki запускается через C#-инструмент репозитория:

```bash
dotnet run --project eng/Electron2D.Build -- update wiki --check
dotnet run --project eng/Electron2D.Build -- update wiki --check --output .github/wiki
dotnet run --project eng/Electron2D.Build -- update wiki --output .github/wiki
```

Команда должна поддерживать два режима:

- обычный режим генерирует Wiki pages в переданный `--output` или во временную папку;
- `--check` без `--output` проверяет, что generator воспроизводимо создаёт expected Wiki pages;
- `--check --output <wiki-clone>` сравнивает expected Wiki pages с рабочей копией `Electron2D.wiki.git`.

`--check` должен завершаться ошибкой, если:

- отсутствует сгенерированная Wiki page;
- сгенерированная Wiki page устарела;
- в GitHub Wiki clone осталась лишняя сгенерированная page;
- отсутствует `API-Compatibility.md` в GitHub Wiki clone.

CI должен клонировать `Electron2D.wiki.git` в `.github/wiki` и запускать `dotnet run --project eng/Electron2D.Build -- update wiki --check --output .github/wiki`.

## Критерии приёмки

- Generator читает compiled public API и XML documentation.
- Для каждого public type создаётся отдельная GitHub Wiki page.
- `Home.md`, `API-by-Category.md`, `API-Reference.md`, `_Sidebar.md` и `_Footer.md` создаются автоматически.
- Для каждого домена API создаётся category page.
- Ссылки между Wiki pages используют wiki-style links без `.md`, чтобы GitHub показывал rendered pages, а не raw Markdown.
- `_Sidebar.md` создаётся автоматически и не выводит весь API surface в навигацию.
- `API-Compatibility.md` сохраняется в `Electron2D.wiki.git`, связана с остальными API reference pages и не заменяется локальным сайтом.
- `dotnet run --project eng/Electron2D.Build -- update wiki --check --output .github/wiki` падает на рассинхронизированной Wiki или устаревшем API manifest и проходит после генерации.
- CI запускает Wiki verifier.
- Документация реализации описывает команды генерации и проверки.

## Фактическое состояние, ограничения и проверки

Текущая реализация генерирует API reference Electron2D в настоящий GitHub Wiki repository:

```text
https://github.com/edwardgushchin/Electron2D.wiki.git
```

Основной репозиторий не отслеживает `.github/wiki/` как часть своего исходного кода и не содержит локальный сайт документации.

Локальный путь `.github/wiki` используется только как рабочий клон GitHub Wiki repository для генерации, проверки и публикации. Этот каталог игнорируется основным репозиторием и не коммитится в него.

## Wiki layout

В `Electron2D.wiki.git` хранятся:

- `Home.md`;
- `_Sidebar.md`;
- `_Footer.md`;
- `API-by-Category.md`;
- `API-*.md` category pages;
- `API-Reference.md`;
- страницы публичных типов с короткими именами, например `ElectronObject.md`, `Node.md`, `Collections-Array.md`;
- `API-Compatibility.md`.

`Home.md`, `_Sidebar.md`, `_Footer.md`, `API-by-Category.md`, category pages, `API-Reference.md`, страницы типов и `API-Compatibility.md` создаются автоматически. Compatibility page содержит generated release/status table из manual profile and generated manifest, не редактируется вручную и должна проходить `verify public-api-documentation --wiki-path .github/wiki`.

## Generator

Команда локальной генерации во временную папку:

```bash
dotnet run --project eng/Electron2D.Build -- update wiki
```

Команда обновления клона GitHub Wiki:

```bash
dotnet run --project eng/Electron2D.Build -- update wiki --output .github/wiki
```

Команда проверки синхронизации с клоном GitHub Wiki:

```bash
dotnet run --project eng/Electron2D.Build -- update wiki --check --output .github/wiki
```

Генератор собирает `src\Electron2D\Electron2D.csproj` с XML documentation file, читает compiled exported public surface из runtime assembly, читает `data/api/electron2d-api-manifest.json` и создаёт GitHub Wiki Markdown pages только для этих exported типов. Ручной список публичных типов не используется; profile-only решения без exported type не материализуются как type pages.

Каждая generated page существующего exported type содержит блок `Godot 4.7 C# profile compatibility` с текущим manifest status и `Parity evidence: not_verified`. Для нормального `Supported` API блок показывает `Supported / Profile approved` и `Out of profile: no`. Решения `deferred` и `unsupported`, как и другие profile-only строки без exported type, остаются в manual profile и проверяются по одному командой `e2d api compare-godot <type>` с сохранением точного статуса; отдельная Wiki type page для них не создаётся.

## Текущий статус

На 2026-07-08 генератор создаёт `Home.md`, `_Sidebar.md`, `_Footer.md`, `API-by-Category.md`, `API-Reference.md`, category pages, страницы публичных типов и generated `API-Compatibility.md`. В `Electron2D.wiki.git` хранится полный API reference для текущего public runtime surface и profile-backed compatibility report.

CI клонирует `Electron2D.wiki.git` в `.github/wiki`, запускает `dotnet run --project eng/Electron2D.Build -- update api-manifest --check`, затем запускает `dotnet run --project eng/Electron2D.Build -- update wiki --check --output .github/wiki`. Если public API, XML documentation, manual API profile, manifest или generated Wiki pages меняются несогласованно, CI падает.

Все внутренние ссылки генерируются без расширения `.md`, чтобы GitHub открывал rendered Wiki pages, а не raw Markdown.

## Что проверяет `-Check`

Verifier сравнивает expected generated Wiki pages с текущими файлами в клоне `Electron2D.wiki.git` и падает, если:

- сгенерированная page отсутствует;
- сгенерированная page устарела;
- осталась лишняя сгенерированная page;
- отсутствует `API-Compatibility.md`.
- tracked `data/api/electron2d-api-manifest.json` устарел относительно public API, XML documentation или manual API profile.

Дополнительный audit проверяет, что `Home.md`, `API-Reference.md`, `_Sidebar.md` и `API-Compatibility.md` связаны wiki-style ссылками без расширения `.md`, а compatibility page содержит generated marker, источник manual profile, status legend и current public runtime surface.

Такой режим гарантирует, что GitHub Wiki остаётся частью обычного review и release gate без отслеживания Wiki pages в основном репозитории.
