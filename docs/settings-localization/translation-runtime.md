## Контракт и ожидаемое поведение

Статус: целевая спецификация для `T-0075`.
Обновлено: 2026-06-21.

## Назначение

Electron2D `0.1-preview` должен иметь минимальный контракт локализации во время выполнения для игр, редактора и будущих настроек проекта. Контракт включает ресурс с переводами, общий сервер переводов, определение начальной локали из окружения процесса, ручную смену локали, перевод через `Tr(string key)` и предсказуемое резервное поведение для отсутствующих ключей.

Для `0.1-preview` обязательные языки пользовательских надписей и примеров ограничены русским и английским. Английский используется как резервный язык `en`; расширение набора языков, региональных вариантов и правил перевода относится к следующим версиям.

## Публичный API

`Translation` наследуется от `Resource` и хранит набор сообщений для одной локали:

- `Locale` - нормализованная строка локали. Дефис в имени локали приводится к подчёркиванию, например `en-US` хранится как `en_US`.
- `AddMessage(string srcMessage, string xlatedMessage, string context = "")` - добавляет или заменяет перевод.
- `EraseMessage(string srcMessage, string context = "")` - удаляет перевод, если он есть.
- `GetMessage(string srcMessage, string context = "")` - возвращает перевод или пустую строку, если ключа нет.
- `GetMessageList()` - возвращает стабильный отсортированный список исходных ключей.

`TranslationServer` управляет загруженными переводами процесса:

- `GetLocale()` возвращает текущую локаль.
- `SetLocale(string locale)` задаёт текущую локаль вручную.
- `AddTranslation(Translation translation)` добавляет ресурс в реестр.
- `RemoveTranslation(Translation translation)` удаляет ресурс из реестра.
- `Clear()` удаляет все загруженные переводы без сброса текущей локали.
- `GetLoadedLocales()` возвращает стабильный отсортированный список локалей, для которых загружены переводы.
- `Translate(string message, string context = "")` возвращает переведённую строку.

`Object.Tr(string message, string context = "")` делегирует перевод в `TranslationServer.Translate()`, поэтому `Node.Tr(string key)` доступен без отдельного API только для узлов.

## Поиск перевода по локали

При старте процесса `TranslationServer` определяет локаль через `CultureInfo.CurrentUICulture`. Пустая или инвариантная культура приводится к `en`. Имена локалей нормализуются только по разделителю `-`/`_`; расширенная таблица языков и стран остаётся отдельной задачей.

Для `Translate()` порядок поиска такой:

1. точная текущая локаль, например `fr_CA`;
2. базовый язык текущей локали, например `fr`;
3. резервная локаль `en`;
4. исходный ключ `message`.

Если `message` пустой, возвращается пустая строка. Если ключ отсутствует во всех загруженных переводах, возвращается исходный ключ, чтобы UI оставался предсказуемым и удобным для отладки.

## Обновление видимого UI

`Label` использует `Tr(Text)` при отрисовке текста. Смена локали или набора загруженных переводов должна сбрасывать кэшированные команды отрисовки для label на следующем обработанном кадре, чтобы видимый текст обновлялся без ручного `QueueRedraw()`.

## Ограничения `T-0075`

- Импорт файлов формата переводов не реализуется в этой задаче.
- Настройки проекта и пользователя для сохранения выбранной локали остаются задачей `T-0076`.
- Формы множественного числа, псевдолокализация и извлечение переводимых строк редактора не входят в эту задачу.
- Сопоставление локалей ограничено точным совпадением, базовым языком и резервной локалью `en` без региональной таблицы.

## Фактическое состояние, ограничения и проверки

Текущая реализация во время выполнения добавляет минимальную модель локализации для `0.1-preview`:

- `Translation` - ресурс с сообщениями для одной локали.
- `TranslationServer` - общий реестр переводов процесса и текущей локали.
- `Object.Tr(string message, string context = "")` - удобный перевод из любого `Object` или `Node`.
- `Label` рисует `Tr(Text)` и обновляет кэшированные команды отрисовки после смены локали или набора переводов.

## `Translation`

`Translation.Locale` хранит нормализованное имя локали. Дефис заменяется на подчёркивание, часть языка приводится к нижнему регистру, часть региона - к верхнему. Например, `pt-BR` становится `pt_BR`.

`AddMessage(srcMessage, xlatedMessage, context)` добавляет или заменяет перевод. Один и тот же исходный ключ может иметь разные переводы для разных значений контекста. `GetMessage()` возвращает перевод или пустую строку, если пара ключа и контекста в этом ресурсе отсутствует. `EraseMessage()` удаляет запись без ошибки, если её нет. `GetMessageList()` возвращает отсортированный список уникальных исходных ключей.

## `TranslationServer`

При первом обращении сервер определяет начальную локаль из `CultureInfo.CurrentUICulture`. Пустое значение приводится к `en`.

`SetLocale()` задаёт текущую локаль вручную. `AddTranslation()` и `RemoveTranslation()` управляют загруженными ресурсами. `Clear()` очищает реестр, но не сбрасывает текущую локаль. `GetLoadedLocales()` возвращает отсортированный список локалей, для которых есть зарегистрированные переводы.

`Translate()` выполняет поиск в таком порядке:

1. точная текущая локаль, например `fr_CA`;
2. базовый язык текущей локали, например `fr`;
3. резервная локаль `en`;
4. исходный ключ.

Такое резервное поведение делает отсутствующие ключи предсказуемыми: UI показывает исходный ключ, а не пустую строку.

## `Label`

`Label.Text` остаётся исходным ключом. Во время `_Draw()` label вызывает `Tr(Text)` и отправляет в canvas уже переведённый текст. `Label._Process()` отслеживает внутреннюю версию состояния переводов и вызывает `QueueRedraw()`, когда локаль или реестр переводов изменились. Поэтому видимый текст обновляется на следующем обработанном кадре без ручного перерисовывания.

## Ограничения

- Импорт файлов переводов не реализован.
- Сохранение выбранной локали доступно во внутреннем файле пользовательских настроек через механизм сохранения настроек; UI выбора локали остаётся задачей редактора.
- Формы множественного числа, псевдолокализация и инструменты извлечения переводимых строк не реализованы.
- Сопоставление локалей ограничено точным совпадением, базовым языком и резервной локалью `en`.
